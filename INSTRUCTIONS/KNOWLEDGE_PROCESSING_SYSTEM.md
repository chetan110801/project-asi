# KNOWLEDGE PROCESSING SYSTEM
**How we turn a resource into real understanding.** Reading is input. This system is the digestion — turning raw material into knowledge that connects, sticks, and can be used.

`Part of: PROJECT ASI — Living Instruction System`
`System Version: 1.0 · Status: Living document · Last updated: 2026-06-20`

---

## 1. The pipeline

```
RESOURCE → EXTRACT → STRUCTURE → EXPLAIN (4 levels) → CONNECT → TEST → STORE
```

A resource that stops at "I read it" is wasted. It only counts once it has passed through to STORE.

---

## 2. EXTRACT — pull out the substance

From each resource, capture:
- **Core ideas** — the 3–7 things that actually matter.
- **Key insights** — the "oh, *that's* how it works" moments.
- **Assumptions** — what the author takes for granted.
- **Dependencies** — what this idea relies on being true.
- **Implications** — what follows if it's right.
- **Contradictions** — where it clashes with other things we believe.
- **Limitations** — where it breaks or doesn't apply.
- **Open questions** — what it leaves unanswered → feed [`OPEN_QUESTIONS_SYSTEM.md`](OPEN_QUESTIONS_SYSTEM.md).

---

## 3. STRUCTURE — organize before explaining

Lay the extracted pieces into a simple skeleton: what's the claim, what supports it, what it depends on, what challenges it. If you can't structure it, you haven't understood it yet — go back.

---

## 4. EXPLAIN — write all four levels

Produce the four explanation levels from [`LEARNING_METHOD.md`](LEARNING_METHOD.md):
- **5-minute** (gist + one analogy + why it matters)
- **30-minute** (parts + how they fit + key example)
- **Deep** (mechanism, details, edge cases)
- **Expert** (open debates, failure cases)

Writing these *is* the understanding. If a level won't write, that's exactly where your understanding has a hole.

---

## 5. CONNECT — link it to everything else

New knowledge floating alone is fragile. We connect it:
- to **other ideas** (agrees with / contradicts / depends on / generalizes),
- to **domains** (which fields it touches),
- to **bottlenecks** (does it ease or worsen a known constraint?),
- to the **AGI/ASI map** (does it change any path's feasibility?).

These connections are drawn in [`MAPS/`] (dependency maps, concept maps) and reflected in [`AGI_ASI_INVESTIGATION_SYSTEM.md`](AGI_ASI_INVESTIGATION_SYSTEM.md).

---

## 6. TEST — prove it stuck

Before calling it understood, pass at least one proof-of-learning from [`LEARNING_METHOD.md`](LEARNING_METHOD.md): explain from memory, draw it, answer a new question, build a toy, or make a prediction. **No proof, no "understood" tag.**

---

## 7. STORE — make it findable and revisitable

Each processed resource becomes an **understanding-note** containing: the four explanation levels, the extracted points, the connections, the proof-of-learning, a confidence tag, and a "revisit on" date for spaced repetition.

Stored well, it can be: re-read at growing intervals, reused in new investigations, and *updated* when new evidence arrives (knowledge is never frozen).

---

## 8. The compounding effect

Done consistently, every new resource gets *easier* to process, because it connects to a richer existing web. Early learning is slow and lonely; later learning is fast because everything has somewhere to attach. This compounding is the whole point of processing instead of just collecting.
